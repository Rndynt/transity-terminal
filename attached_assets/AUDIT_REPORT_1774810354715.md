### [C-01] ⚡ Kredensial Produksi Hardcoded di `.env` yang di-Commit

**File:** `.env`  
**Tipe:** Security — Exposed Secrets  

```env
DATABASE_URL=postgresql://transity:transity123@localhost:5432/transitycore
DEV_BYPASS_AUTH=true
```

Lebih parah: baris 6 berisi konkatenasi yang rusak:
```
PENDING_BOOKING_AUTO_RELEASE=trueDEV_BYPASS_AUTH=true
```

Kedua variabel ini **tidak terbaca dengan benar** oleh parser env. `DEV_BYPASS_AUTH` tidak akan pernah aktif, tapi juga tidak terproteksi. File `.env` harus segera dimasukkan ke `.gitignore` dan kredensial dirotasi.

**Perbaikan:** Hapus `.env` dari git, tambahkan ke `.gitignore`, gunakan hanya `.env.example` tanpa nilai asli.

### [C-02] ⚡ Transaction Escape — DB Writes Keluar dari Blok Transaksi

**File:** `server/modules/bookings/bookings.service.ts` (baris 188–242)  
**Tipe:** Logic Bug / Data Integrity  

```typescript
const booking = await db.transaction(async (tx) => {
  const newBooking = await this.storage.createBooking({ ... });   // ❌ pakai koneksi sendiri
  await this.storage.createPassenger({ ... });                     // ❌ pakai koneksi sendiri
  await tx.update(seatInventory)...                                // ✅ benar
  await this.storage.createPayment({ ... });                       // ❌ pakai koneksi sendiri
  await this.storage.createPrintJob({ ... });                      // ❌ pakai koneksi sendiri
});
```

Jika `tx.update(seatInventory)` gagal, booking/passenger/payment **sudah commit** di DB tanpa bisa di-rollback. Menghasilkan **ghost booking berbayar dengan seat hold yang tidak dibersihkan**.

**File:** `server/modules/bookings/reschedule.service.ts` (baris 42–56)
